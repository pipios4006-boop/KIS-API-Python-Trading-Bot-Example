🔹 매도 데드존(Dead Zone) 및 잔량 누적 이월(Carry-over) : VWAP 타임 슬라이싱 매도 시 현재 시장 호가가 목표가 미만일 경우 매도를 강제 동결(데드존). 미체결 슬라이싱 물량은 소멸하지 않고 다음 1분 사이클로 누적 이월되어, 목표가 회복 시 잔량까지 통합 타격(Sweep) 집행.
🔹 잭팟 & 1층 잔량 강제 청산 (Sweep Finisher) : 장 마감 2분 전(15:58 EST), 잭팟 커트라인(1.010배) 돌파 시 전량 덤핑. 추가로 주가가 1층 타점(1.006배) 이상을 유지 중이면 1층 잔여 물량을 시장 매수호가(Bid Price)로 전량 덤핑하여 1층 지층 완벽 소각 (타점 미달 시 익일 큐로 안전하게 이월).
🔹 애프터마켓 3% 로터리 덫 (Lottery Trap) : 정규장 마감 직후(16:05 EST / 05:05 KST) 잔여 물량 100%를 총 평단가 대비 +3% 타점에 '장후 지정가(AFTER_LIMIT)'로 셋팅. 유동성 공백과 실적 발표 노이즈를 활용한 시간 외 폭등 포획 전술 (미체결 시 익일 오전 자동 소멸).
🔹 3중 경고 장부 초기화 시스템 : V-REV 가동 중 장부 초기화 시 다단 지층이 붕괴되는 파괴적 조건을 명시한 UI 경고창 출력 후, 최종 승인 시에만 수동 이관(INIT) 압축 실행.
🔹 실시간 공수 교대 및 긴급 수혈 (Emergency) : VWAP 타임 슬라이싱 중 앵커 돌파 시 0.1초 만에 매수/매도 태세 전환. 장 마감 직전 익일 매수 예산(0.5회분)이 부족할 경우 가장 최근 로트를 MOC 강제 매도하여 실탄(현금) 우선 확보.
🛡️ **[ 다중 종목 에스크로 및 예산 고갈 방어 로직 ]**
🟢 휘발성 가상 장부 (Escrow) 락다운 : 특정 종목 리버스 진입 시 현금을 가상 장부(Escrow)에 격리.
🟢 과거 꼬리표 얽힘 버그 픽스 : 장부를 역순 탐색하여 '연속된 최근 리버스 기록'만 추출해 에스크로 잔금 동적 산출.
🟢 예산 소진 시 동적 탈출 : 예산 1회분 미만이거나 4주 미만 시 잔량 청산 및 의무 수혈(MOC) 매도로 생명수 창출.
🟢 제논의 역설 완벽 방어 : 에스크로 잔액에 의존하지 않고 리버스 기간 내 '순수 누적 매도 원금'만을 역산 도출하여 고정 4분할 타격 예산 앵커링.
🟢 리버스 멱등성(Idempotency) : 지시서 수시 조회 시 상태가 허위로 덮어씌워지지 않도록 작전 엔진 내 Write 권한 원천 차단.
## 📜 **[ 방탄 TrueSync & 데이터 무결성 절대 원칙 ]**
🟢 로직 A [초기 진입] : 장부가 비어있으나 한투에 수량이 있으면 [INIT 스냅샷]으로 첫 줄 기입.
🟢 로직 B [수익 졸업 및 마이너스 손절 개방] : 한투 0주 + 장부 기록 시 총 수익금이 플러스(+)면 명예의 전당 박제 및 자동 복리 증액. 마이너스(-) 수익 상태라도 0주 팩트 확인 시 락다운을 풀고 즉각 장부를 비워 0주 새출발 디커플링 모드로 강제 리셋.
🟢 로직 C [TrueSync 비파괴 업데이트]
📝 Fast Track : 총수량이 일치하면 합격. 평단가 미세 오차는 한투 팩트(actual_avg)로 교정 덮어쓰기.
📝 비파괴 보정 (CALIB) : 수량 오차 발생 시 과거 역사를 100% 보존한 채 부족/초과된 수량만큼만 '비파괴 보정(CALIB)' 행 추가. (전체 덮어쓰기 절대 금지)
✂️ 액면분할 자동 감지 : 야후 연동으로 액면 분할/병합 시 과거 장부 100% 소급 비율 조정.
⏳ 주말 시차 맹점 방어 : 주말/휴장일에도 최근 거래일의 팩트 종가(prev_close)를 신뢰하도록 강제.
🟢 로직 D [삼위일체 소각 엔진] : /reset 수동 격발 시 본장부, 가상장부(Escrow), 지층 큐(Queue), 숨겨진 백업본까지 4중 찌꺼기 데이터를 100% 동시 소각하여 좀비 장부 부활 엣지 케이스 원천 차단.
🟢 로직 E [텔레그램 API 통신 방어] : 글로벌 네트워크 지연 및 서버 병목 현상에 대비하여 봇 구동 시 read/write/connect/pool 타임아웃을 30초로 강제 확장하고 커넥션 풀을 512로 증설하여 시스템 교착 상태(Deadlock) 원천 차단.
🟢 로직 F [동적 RP 권장액 차감 로직] : telegram_view 시현 시, LOC 주문 체결 전(is_locked=False)에는 필수 예산을 선 차감하여 표기하고, 주문 전송 후에는 삭감된 계좌 잔고를 반영하여 이중 차감 버그를 방어하는 자율주행 디스플레이 보장.
🟢 로직 G [의존성 주입(DI) 완비] : TelegramController 생성 시 vwap_strategy, queue_ledger, strategy_rev 모듈을 완벽 주입하여 명령어 라우팅 시 메모리 크래시 방어.
